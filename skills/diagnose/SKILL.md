# 진단 (Diagnose)

목표: 바로 고치지 말고 먼저 재현한다.

Matt식 진단(diagnose)의 작은 버전이다. 가볍지만 재현과 확인은 생략하지 않는다.

## 진행 절차 (Workflow)

1. **기대 결과(Expected)**: 사용자가 기대한 결과를 한 문장으로 쓴다.
2. **실제 결과(Actual)**: 실제로 본 결과를 한 문장으로 쓴다.
3. **재현 단계(Repro Steps)**: 재현 단계를 번호 목록(numbered list)으로 만든다.
4. 직접 실행해서 재현한다. 웹 UI(web UI)면 브라우저(browser)를 열고 확인한다.
5. 가능한 원인 2-3개를 우선순위 목록(ranked list)으로 적는다.
6. 가장 작은 수정 하나를 한다.
7. 같은 재현 단계(repro steps)로 다시 확인한다.
8. 후속 작업이 필요하면 TODO 후보를 제안한다. `TODO.md`는 사용자가 명시적으로 요청할 때만 수정한다.

## 모르는 것이 있으면

한 번에 하나만 묻는다.

- 어떤 버튼을 눌렀는가?
- 기대한 화면은 무엇인가?
- 실제로 무엇이 보였는가?
- 오류 메시지(error message)가 있는가?

## 금지

- 재현 없이 코드를 고치지 않는다.
- 여러 원인을 한 번에 고치지 않는다.
- `TODO.md`를 조용히 수정하지 않는다.
- 스택 트레이스(stack trace)만 보여주고 멈추지 않는다.
- 사용자가 코드를 읽어야만 이해할 수 있게 설명하지 않는다.

## 완료 형식

```markdown
## 진단 완료 (Diagnosed)

**기대 결과(Expected):** ...
**실제 결과(Actual):** ...
**재현(Repro):** ...
**원인(Cause):** ...
**수정(Fix):** ...
**확인한 것(Checked):** ...
**아직 확인하지 못한 것(Still not checked):** ...
```
